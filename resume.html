자기소개

안드로이드 개발자로서 8년 이상의 경력을 보유하고 있으며, Kotlin과 Jetpack Compose를 활용한 최신 앱 개발에 능숙합니다. MVVM 아키텍처와 Coroutine을 활용하여 안정적이고 효율적인 앱을 개발해왔으며, Web3 지갑 및 블록체인 서비스를 개발하며 Ethereum과 Bitcoin을 포함한 다양한 블록체인 네트워크를 연동하고, MPC(Multi-Party Computation) 기술을 활용하여 사용자 지갑 보안성을 강화한 경험이 있습니다. 또한, GitHub Actions를 활용한 CI/CD 자동화 구축 경험을 통해 배포 효율성을 높이고 코드 품질을 유지해왔습니다. 기획자, 디자이너, 서버 개발자와의 긴밀한 협업을 통해 일정 준수 및 원활한 커뮤니케이션을 지향합니다.

  • Jetpack Compose를 도입하며 LazyColumn과 remember 등을 통해 불필요한 재컴포지션을 최소화하고, Coil을 이용한 이미지 최적화로 약 30% UI 렌더링 성능 향상을 달성했습니다. JankStats를 활용하여 렌더링 지연 구간을 수치로 분석하고 개선했습니다.
  • AsyncTask와 콜백 기반의 비동기 로직을 Coroutine과 viewModelScope로 전환하고, 네트워크 및 데이터베이스 호출은 Dispatchers.IO에서, UI 갱신은 Dispatchers.Main에서 처리하도록 명확히 분리했습니다. 각 코루틴 블록에 try-catch를 적용해 예외를 안전하게 처리하여 앱 크래시를 방지했으며, 그 결과 Firebase 기준 크래시율을 약 20% 감소시켰습니다.
  • GitHub Actions를 활용한 CI/CD 자동화 구축 경험을 통해 배포 효율성을 높이고 코드 품질을 유지해왔습니다.
  • React Native 사이드 프로젝트를 통해 크로스플랫폼 앱 개발 경험을 보유하고 있습니다.
  • 기획자, 디자이너, 서버 개발자와의 긴밀한 협업을 통해 일정 준수 및 원활한 커뮤니케이션을 지향합니다.

아키텍처 및 설계
  • App Architecture 기반으로 UI, Domain, Data Layer를 명확히 분리하고, DIP(Dependency Inversion Principle)에 따라 각 계층의 의존성을 인터페이스로 추상화.
    UI 레이어는 도메인 계층(UseCase)만 참조, Repository/RemoteDataSource 등은 Hilt로 의존성 주입하여 테스트 및 유지보수 용이.
  • Jetpack Compose를 활용한 선언적 UI 구현. @Stable·@Immutable 애너테이션을 적용해 불필요한 recomposition 최소화.
    LazyColumn, remember, derivedStateOf, key 등 Compose 성능 최적화 API 적극 활용. UI 상태는 ViewModel의 StateFlow로 관리, recomposition trace 및 JankStats로 프레임 드랍 구간을 정량 분석·개선.
  • Coroutine 기반 비동기 처리 최적화.
    기존 AsyncTask/Callback 기반 로직을 Coroutine 및 viewModelScope로 이관, try-catch, Dispatcher 분리 등으로 앱 안정성 및 응답성 향상.
  • Room Database로 데이터 캐싱 및 오프라인 지원.
    API 결과를 Room에 저장, 화면은 Flow로 최신 캐시/Remote 데이터 자동 갱신. 네트워크 실패 시 캐시 fallback으로 UX 안정성 확보.
  • JankStats, Layout Inspector 등 진단 도구 적극 도입.
    Compose recomposition, Room 쿼리, 네트워크 IO 등 병목 구간을 측정·개선. Stable/Immutable 애너테이션, recomposition trace, 프레임 드랍 모니터링 등 고도화.
  • 보안 및 민감 데이터 처리: 프라이빗키·시드 등 민감 정보는 Android KeyStore에서 직접 관리, 서명 연산은 Hash 데이터로만 수행하여 외부 노출 원천 차단.
    지갑 생성/복구/서명/인증 등 모든 민감 데이터는 Biometric 인증·KeyStore 기반 암호화로 보호. MPC 프로토콜로 키 분산 저장, 서버와 end-to-end 암호화 통신.
  • 블록체인 트랜잭션 처리: EVM 계열(GasPrice, GasLimit, Nonce) 계산을 네트워크 상태/잔고 기반으로 동적 산정. Bitcoin, Aptos 등 네트워크별 트랜잭션 서명 로직 별도 모듈화로 확장성·유지보수성 강화.
  • 성능 및 보안 최적화: Compose recomposition 최소화, 네트워크 IO 분리, JankStats, Proguard/R8 등 다양한 진단·난독화 도구 통합.
    최근에는 Compose의 @Stable/@Immutable 적용, recomposition trace, JankStats 프레임 드랍 모니터링, SQL 인덱스 최적화 등 고도화 진행.
  • 테스트 코드 구현: JUnit, Mockito 을 활용한 단위 테스트 및 Espresso를 활용한 UI 테스트 설계 및 구현.
    비즈니스 로직 검증을 위한 UseCase 단위 테스트, ViewModel 테스트 구현. Repository 계층의 Mock 객체를 활용해 네트워크 의존성 없이 테스트 가능한 구조 설계. Github Actions CI/CD 파이프라인에 테스트 자동화 통합으로 코드 품질 보장.

UI Layer
│
├── MainActivity (UI 진입점)

│   │

│       └── UseCase 호출 (Domain 계층 인터페이스)
│
Domain Layer
│
├── Use Cases (비즈니스 로직)
│   ├── GetUserSettingsUseCase
│   ├── AuthUseCase
│   ├── PairWalletUseCase
│   ├── BillingUseCase
│   └── 기타 UseCase
│
Data Layer
│
├── Repository (인터페이스 추상화, DIP 원칙 적용)
│   ├── UserDataRepository
│   ├── AuthRepository
│   ├── NetworkRepository
│   ├── BillingRepository
│   └── WalletRepository 등
│
└── Data Sources (Local & Remote)
    ├─ Remote Data Source
    │  ├── Retrofit API 서비스 (서버 데이터)
    │  ├── Firebase Cloud Messaging (FCM)
    │  └── Google BillingClient (결제 서버)
    │
    └─ Local Data Source
        ├── NetworkDao (Room 데이터베이스)
        ├── ProtoDataStore (로컬 저장소)
        └── Security 모듈 (KeyStore, Biometric, MPC)

자기소개

안드로이드 개발자로서 8년 이상의 경력을 보유하고 있으며, 사용자 관점에서 불편함을 발견하고 기술적 해결책을 제시하는 것에 중점을 두고 있습니다. 앱 크래시율을 20% 감소시켜 사용자 불편함을 크게 줄였고, UI 렌더링 성능을 30% 향상시켜 사용자가 더 매끄러운 앱을 경험할 수 있도록 했습니다. 복잡한 암호화폐 거래에서 발생하는 기술적 문제점들을 파악하고 개선 방안을 제안하여 일반 사용자도 쉽게 블록체인 기술을 활용할 수 있도록 했습니다. App Architecture와 MVVM 패턴을 통해 코드 품질을 확보하고, Jetpack Compose와 Coroutine을 활용해 사용자 응답성을 높였습니다. 멀티플랫폼 개발 경험(React Native, iOS, 웹)을 통해 다양한 환경에서 일관된 사용자 경험을 제공하며, GitHub Actions 자동화로 빠른 배포와 안정적인 서비스 운영을 보장합니다.

  • 사용자 불편 사항 발견 및 기술적 해결: 복잡한 암호화폐 거래 과정에서 사용자들이 겪는 어려움을 분석하고, 기존 5단계 거래 절차를 3단계로 단순화할 수 있는 기술적 개선안을 제안. API 호출 최적화와 상태 관리 개선으로 사용자 실수를 90% 줄였습니다.
  • 성능 최적화로 사용자 불편 해소: Jetpack Compose와 @Stable/@Immutable 애너테이션을 통해 불필요한 재컴포지션을 방지하고, 이미지 최적화로 30% UI 렌더링 성능 향상. 사용자들이 느끼던 앱 버벅거림 현상을 크게 개선했습니다.
  • 안정성 확보로 신뢰성 향상: AsyncTask 기반 로직을 Coroutine으로 전환하고 예외 처리를 강화하여 앱 크래시율을 20% 감소. 사용자가 중요한 거래 중 앱이 종료되는 불편함을 해결했습니다.
  • 기술적 복잡성 해결: 복잡한 블록체인 기술을 일반 사용자도 사용할 수 있도록 백그라운드 처리 로직 개선과 에러 핸들링 강화를 제안. 기술적 오류 메시지를 사용자 친화적 메시지로 변환하는 시스템 구현으로 사용자 만족도를 크게 향상시켰습니다.
  • 빠른 문제 해결: Clean Architecture와 모듈화 설계로 버그 수정 시간을 단축하고, GitHub Actions 자동화로 핫픽스 배포 시간을 기존 1시간에서 10분으로 단축하여 사용자 불편을 최소화했습니다.
  • 사용자 피드백 반영: 앱 스토어 리뷰와 사용자 피드백을 적극 반영하여 지속적인 UX 개선. 평점을 4.1에서 4.5으로 향상했습니다.

아키텍처 및 설계
  • App Architecture 기반으로 UI, Domain, Data Layer를 명확히 분리하고, DIP(Dependency Inversion Principle)에 따라 각 계층의 의존성을 인터페이스로 추상화.
    UI 레이어는 도메인 계층(UseCase)만 참조, Repository/RemoteDataSource 등은 Hilt로 의존성 주입하여 테스트 및 유지보수 용이.
  • Jetpack Compose를 활용한 선언적 UI 구현. @Stable·@Immutable 애너테이션을 적용해 불필요한 recomposition 최소화.
    LazyColumn, remember, key 등 Compose 성능 최적화 API 적극 활용. UI 상태는 ViewModel의 StateFlow로 관리, recomposition trace 및 JankStats로 프레임 드랍 구간을 정량 분석·개선.
  • Coroutine 기반 비동기 처리 최적화.
    기존 AsyncTask/Callback 기반 로직을 Coroutine 및 viewModelScope로 이관, try-catch, Dispatcher 분리 등으로 앱 안정성 및 응답성 향상.
  • Room Database로 데이터 캐싱 및 오프라인 지원.
    API 결과를 Room에 저장, 화면은 Flow로 최신 캐시/Remote 데이터 자동 갱신. 네트워크 실패 시 캐시 fallback으로 UX 안정성 확보.
  • JankStats, Layout Inspector 등 진단 도구 적극 도입.
    Compose recomposition, Room 쿼리, 네트워크 IO 등 병목 구간을 측정·개선. Stable/Immutable 애너테이션, recomposition trace, 프레임 드랍 모니터링 등 고도화.
  • 보안 및 민감 데이터 처리: 프라이빗키·시드 등 민감 정보는 Android KeyStore에서 직접 관리, 서명 연산은 Hash 데이터로만 수행하여 외부 노출 원천 차단.
    지갑 생성/복구/서명/인증 등 모든 민감 데이터는 Biometric 인증·KeyStore 기반 암호화로 보호. MPC 프로토콜로 키 분산 저장, 서버와 end-to-end 암호화 통신.
  • 블록체인 트랜잭션 처리: EVM 계열(GasPrice, GasLimit, Nonce) 계산을 네트워크 상태/잔고 기반으로 동적 산정. Bitcoin, Aptos 등 네트워크별 트랜잭션 서명 로직 별도 모듈화로 확장성·유지보수성 강화.
  • 성능 및 보안 최적화: Compose recomposition 최소화, 네트워크 IO 분리, JankStats, Proguard/R8 등 다양한 진단·난독화 도구 통합.
    최근에는 Compose의 @Stable/@Immutable 적용, recomposition trace, JankStats 프레임 드랍 모니터링 등 고도화 진행.
  • 테스트 코드 구현: JUnit, Mockito 을 활용한 단위 테스트 및 Espresso를 활용한 UI 테스트 설계 및 구현.
    비즈니스 로직 검증을 위한 UseCase 단위 테스트, ViewModel 테스트 구현. Repository 계층의 Mock 객체를 활용해 네트워크 의존성 없이 테스트 가능한 구조 설계. Github Actions CI/CD 파이프라인에 테스트 자동화 통합으로 코드 품질 보장.

App Architecture Layered 

UI Layer
│
├── MainActivity (UI 진입점)
│   ├── ABCApp (Compose 루트)
│   │   └── ABCNavHost (네비게이션)
│   │       ├── HomeScreen
│   │       ├── AssetScreen
│   │       └── 기타 Screens...
│   │
│   └── MainViewModel (UI 상태 관리, Flow 기반)
│       └── UseCase 호출 (Domain 계층 인터페이스)
│
Domain Layer
│
├── Use Cases (비즈니스 로직)
│   ├── GetUserSettingsUseCase
│   ├── AuthUseCase
│   ├── PairWalletUseCase
│   ├── BillingUseCase
│   └── 기타 UseCase
│
Data Layer
│
├── Repository (인터페이스 추상화, DIP 원칙 적용)
│   ├── UserDataRepository
│   ├── AuthRepository
│   ├── NetworkRepository
│   ├── BillingRepository
│   └── WalletRepository 등
│
└── Data Sources (Local & Remote)
    ├─ Remote Data Source
    │  ├── Retrofit API 서비스 (서버 데이터)
    │  ├── Firebase Cloud Messaging (FCM)
    │  └── Google BillingClient (결제 서버)
    │
    └─ Local Data Source
        ├── NetworkDao (Room 데이터베이스)
        ├── ProtoDataStore (로컬 저장소)
        └── Security 모듈 (KeyStore, Biometric, MPC)
